# Career RPG 開發進度總結 (Developer Handoff Note)

這份文件總結了近期 Career RPG 專案在前端介面與流程邏輯上的核心修改，可直接作為新 AI 對話的背景 Context 使用，幫助新的 AI 快速掌握專案現況。

## 1. 視覺與互動優化 (UI/UX & Animations)
- **開場雲霧動畫 (Soft Float & Blur)**：
  - 移除了中央遮擋視線的雲層 (`.cloud-5`)。
  - 將四個角落的雲朵 (`.cloud-1` ~ `.cloud-4`) 往外推至 `-40%` 的位置，並統一將不透明度 (opacity) 調整為 `0.95`，確保畫面通透且維持四角的視覺平衡。
  - **修復按鈕誤觸 Bug**：在按鈕的 CSS Keyframes 轉場動畫中加入了 `pointer-events: none` 到 `auto` 的漸變控制，確保按鈕必須在動畫完全浮現後才能被點擊。
- **頁面轉場與載入**：
  - 放棄傳統的 slide 滑動，全站改採 Gaussian Blur（高斯模糊）的柔和淡入淡出效果 (`opacity` + `filter: blur`)，提升沉浸感。
  - 修正了重新整理頁面時的畫面閃爍問題：在 `window.onload` 觸發 `goTo()` 時加入 `immediate` 參數，略過首次載入的退場動畫，強制正確渲染首頁。
- **字體與閱讀性**：
  - 標題全面改用陶土色（`#E07A5F`），並加上白色光暈（`text-shadow`），確保在任何複雜的雲層背景下皆具備高對比度與高閱讀性。

## 2. 業務邏輯與流程分流 (Routing & Logic Refactor)
依據流程圖 (2-1 想不想分流、2-2 能不能分流) 對 `flow-pages.html` 進行了深度的結構重構：

- **雙軌流程分離**：將原本單一的流程拆分為「想不想 (Want)」與「能不能 (Can)」兩條獨立路線。
- **「想不想」專屬路線建置 (Flowchart 2-1)**：
  - 新增專屬頁面：`page-event-summary-want` (事件簡述 1)、`page-adjust-want` (調整向度/增加細節)、`page-event-summary-2-want` (事件簡述 2)。
  - **Q1~Q3 雙向綁定 (Two-way Binding)**：將「調整向度」頁面替換為三個獨立的 Q1~Q3 輸入框。透過 Javascript 與最初問卷資料連動，修改時會自動同步舊資料，事件簡述也會即時抓取最新問卷結果。
  - **流程串接**：點選「符合 / 沒想法」或「先試試看吧～」按鈕，會自動將流程導向「能不能」路線的首頁 (`page-can-or-not`)。
- **「能不能」路線微調 (Flowchart 2-2)**：
  - 遵循流程圖指示：將按鈕「符合 / 沒問題」更新為「符合 / 沒想法」；將調整頁面的標題更新為「調整向度 / 增加細節」，並加上副標題「（其他條件不變）」。
- **排除未實作功能 (暫存區)**：
  - 刻意跳過實作「What we had known about you」系列頁面與流程圖中紅圈標示的進階下拉式選項。

## 3. 表單輸入體驗升級 (Form & Input UX)
- **多行輸入框 (Textarea)**：
  - 為了解決預設提示詞（Placeholder）過長導致單行文字框被截斷的問題，將所有的問卷 (`Q1~Q3`)、事件描述 (`input-can-or-not`) 及調整向度 (`input-adjust`) 的 HTML 標籤從 `<input type="text">` 全面升級為 `<textarea>`。
  - 預設高度設定為約 3 行 (`min-height: 80px`)，確保提示詞自動換行完整顯示，並提供使用者更寬裕的書寫空間。
- **預設引導提示詞 (Placeholder)**：
  - **Q1**: 說出一件「不論別人如何評價、評價如何，但你對結果很滿意」的經歷
  - **Q2**: 說出一件，「不是單純出於善意，但就算 沒有 實質收穫／獎勵，你仍然願意付出努力」的事
  - **Q3**: 說出一件「不論別人如何評價、評價如何，但你對結果很滿意」的經歷
  - *以上提示詞在初始問卷與「調整向度」頁面中皆會同步顯示。*
- **字數限制 (Maxlength)**：
  - 為這 8 個 `<textarea>` 加入了 `maxlength="300"` 的原生屬性限制，確保內容不超過 300 字。
  - **(隱藏限制)** 依需求，刻意不加入任何「x/300」的視覺計數器或警告提示，保持畫面設計的極簡與乾淨。

## 4. 當前檔案架構狀態

- `flow-pages.html`: 負責所有的問卷互動、狀態管理 (LocalStorage) 以及跳轉邏輯。
